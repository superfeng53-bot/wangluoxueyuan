# 常驻自动任务服务 — 成都职业培训网络学院（公需年度型）

> **site_profile: B** — 账号配置目标年度，按年串行完成培训视频学习。

## 1. 文档目的

1. 单账号：登录 → `target_years` 顺序 → 每年 `run_year_task` → `my_train.progress` 达标。
2. 状态：`year_status`、课程章节 `state/progress`。
3. 多账号并行；单账号内年度与章节串行。
4. Web：近 5 年 pill、按年进度（Phase 5）。

## 2. 站点参数

| 项 | 值 |
|----|---|
| 平台 | 成都职业培训网络学院 |
| 站点 | https://www.wlxy.org.cn/ |
| API | https://api.cdwork.cn |
| 年度来源 | `GET /train/train/my_train` |
| 年课表 | `GET /train/train/train_resource_list` |
| 学习上报 | `POST /train/course/start_learning` |
| 年度完成 | `my_train.progress>=1` 或 `state==3` |
| 标准上报间隔 | 15s |
| 快速上报间隔 | 3s |
| 未报名年度 | `fail`（不在 my_train） |

## 3. 核心业务

- **target_years_json**：如 `["2026","2025"]`；当前年优先，其余降序。
- **无**学科槽位、无申请学分队列。
- **考试**：平台有 `/train/exam/*`；公需测试包内未见考试课，待考列表为空时跳过。

## 4. 单账号循环

1. 会话恢复（cookies + `get_userByToken`）
2. `for year in target_years`: `YearTaskRunner.run_year_task(year)`
3. 年内：过滤未完成章节 → 串行 `study_chapter`
4. 全部完成 → `completed`

**无** `MAX_LEARN_PER_DAY`、无 `waiting_apply`。

## 5. CLI 验证

```bash
python3 -m wlxy_api.cli_train years
python3 -m wlxy_api.cli_train year 2026 --chapters
python3 -m wlxy_api.cli_year --year 2026 --dry-run
```
